# Кодогенерация дерева свойств (редактор)

Краткая **пользовательская** справка: как помечать поля в заголовках, какие типы поддерживаются и какие ключи в `/// @property` / `/// @getter` / `/// @setter` влияют на инспектор.

Технические детали сканера и ограничения v1 см. также [`.cursor/rules/property-codegen.mdc`](../.cursor/rules/property-codegen.mdc). Реализация UI: `src/Engine/Editor/PropertyTreeDrawer.cpp`, структура подсказок: `src/Engine/Core/PropertyMeta.h`.

---

## Зачем это нужно

Для классов с `META_CLASS()` генератор создаёт `src/Codegen/<ИмяФайла>.generated.hpp` с реализацией `BuildPropertyTree` — дерево полей для панели свойств редактора. Сборка запускает `tools/property_codegen.py` (цель CMake **`Codegen`**; полная перегенерация — **`Codegen_Force`** или `python tools/property_codegen.py --root . --force`).

---

## Минимальный порядок действий

1. В **заголовке** класса: `#include "Engine/Core/MetaClass.h"`, внутри класса — `META_CLASS()`.
2. Наследник с деревом базы: на отдельной строке **`META_PROPERTY_BASE(ImmediateBase)`** (один раз, рядом с `META_CLASS()`).
3. Перед полем или геттером — однострочный комментарий `/// @property(...)` или `/// @getter(...)` / `/// @setter(...)`.
4. В **одном** соответствующем `.cpp` после включения своего `.h`: `#include "<Имя>.generated.hpp"` (подключается сгенерированный заголовок из `src/Codegen/`).

Идентификатор узла в дереве выводится из имени поля или метода; подпись в UI — из `name`/`label` или по правилам вывода из имени (см. правило выше).

---

## Ключи в тегах (`@property`, `@getter`, `@setter`)

Синтаксис: `/// @property(key=value, ...)` — одна строка; следующая непустая строка кода — объявление поля или метода, которое сканер сопоставляет с тегом.

| Ключ в теге | Где допускается | Назначение |
|-------------|-----------------|------------|
| `name` | `@property`, `@getter`, `@setter` | Подпись в инспекторе (и сопоставление геттер/сеттер по строке). |
| `label` | то же | То же, что `name` (альтернативное имя ключа). |
| `readonly` | то же | `true` — только чтение; для `@getter` без сеттера свойство всегда только чтение. |
| `setter` | только `@property` для поля типа **`bool`** | Имя метода `this->Method(bool)` вместо прямой записи в член. |
| `setter` | `@getter` | Альтернатива отдельному `@setter`: имя метода для записи значения. |
| `tooltip` | то же | Текст подсказки (строка в кавычках). |
| `minValue`, `maxValue` | то же | Число → подсказки диапазона в `PropertyMeta` (см. таблицу типов ниже: не для всех типов используются в UI). |
| `dragSpeed` | то же | Скорость drag для `float`/`double`/`sf::Vector2f`/`sf::Vector3f` и шаг для `DragInt2`/`DragScalarN` у целочисленных/беззнаковых пар. |
| `minCount`, `maxCount` | только поля **`std::vector<…>`** | Ограничение размера списка кнопками +/- в инспекторе. |

Слияние мета с пары `@setter` → `@getter`: ключи вроде `dragSpeed`, `minValue`, … на сеттере подмешиваются к свойству, если на геттере того же ключа нет.

## Поддерживаемые типы (v1)

### Скаляры и `sf::` (поле или тип элемента `std::vector` / ключа и значения `map` / элемента `set`)

| Тип C++ | Инспектор (кратко) |
|---------|-------------------|
| `float` | Drag float |
| `double` | Drag double |
| `bool` | Чекбокс; опционально `setter=ИмяМетода` |
| `int`, `std::int32_t` | Drag int |
| `std::int64_t` | Drag 64-bit |
| `std::string` | Однострочное поле ввода |
| `sf::Vector2f` | Два float |
| `sf::Vector2i` | Два int (`DragInt2`), общие min/max на обе оси |
| `sf::Vector2u` | Два unsigned 32-bit |
| `sf::Vector3f` | Три float |
| `sf::Color` | `ColorEdit4` |
| `sf::Angle` | **Градусы** в UI (`addFloat`: значение с геттера/в поле — через `asDegrees()` / запись через `sf::degrees`) |

### Коллекции

| Тип | Ограничения v1 |
|-----|----------------|
| `std::vector<T>` | `T` — любой скаляр из таблицы выше. Доступны `@getter`/`@setter` с тем же `T` или вектором. |
| `std::map<K,V>`, `std::unordered_map<K,V>` | Только **члены данных**, не `@getter`. `K` и `V` — из таблицы скаляров. |
| `std::set<E>`, `std::unordered_set<E>` | То же; `E` — скаляр из таблицы. |
| `std::bitset<N>` | Только член; фиксированная длина, по индексу bool. Не `@getter`. |

## Какие мета-ключи реально влияют на UI (по типам)

Ниже — только то, что **использует** текущий `PropertyTreeDrawer` при отрисовке.

| Тип | `readonly` | `tooltip` | `minValue` / `maxValue` | `dragSpeed` | `minCount` / `maxCount` |
|-----|:---:|:---:|:---:|:---:|:---:|
| `float`, `double` | да | да | да | да | — |
| `sf::Angle` (как float) | да | да | да | да | — |
| `int`, `std::int32_t`, `std::int64_t` | да | да | нет | нет | — |
| `bool` | да | да | нет | нет | — |
| `std::string` | да | да | нет | нет | — |
| `sf::Vector2f`, `sf::Vector3f` | да | да | нет | да | — |
| `sf::Vector2i` | да | да | да (обе оси) | да (чувствительность `DragInt2`) | — |
| `sf::Vector2u` | да | да | да (как `uint32_t`, обе оси) | да | — |
| `sf::Color` | да | да | нет | нет | — |
| `std::vector<T>` (узел списка) | да | да | см. тип `T` для **каждого элемента** | см. `T` | да (размер списка) |
| `std::map` / `std::set` / … | да | да | нет* | нет* | нет* |
| `std::bitset<N>` | да | да | нет | нет | нет |

\*Для ассоциативных контейнеров в сгенерированном коде у вложенных полей «ключ»/«значение» передаётся пустая `PropertyMeta`; у корня контейнера — мета из тега (в основном `readonly` и `tooltip`). Числовые подсказки на корне для map/set в типичном UI не используются.

---

## Что кодогенератор **не** поддерживает (v1)

- Многострочный ввод строки через тег (`PropertyMeta::stringMultiline` в C++ есть, генератор его не выставляет).
- `@getter` / `@setter` для `std::map`, `std::set`, `std::bitset`.
- Вложенные `std::vector`, пользовательские структуры как `T`, шаблоны с «сложными» аргументами в `map`/`set`.
- Полный разбор C++: только задокументированный подмножество синтаксиса одной строки на объявление.
